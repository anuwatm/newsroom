# Newsroom System MVP 📰

ระบบเขียนและจัดเตรียมบทข่าว (Newsroom Script Editor)  ถูกออกแบบให้ใช้งานง่าย รวดเร็ว และรองรับการพิมพ์บทข่าวสำหรับให้ทีมงานและผู้ประกาศข่าวอ่านได้อย่างมีประสิทธิภาพ

## 🌟 ฟีเจอร์หลัก (Key Features)

1. **ระบบบทข่าว 2 คอลัมน์ (Dual-Column Editor)**
   - **ฝั่งซ้าย (Production Cues):** คำสั่งเทคนิคสำหรับทีมผลิต (Director, CG, Audio) ในรูปแบบบล็อกข้อมูล ประกอบไปด้วยรายการคำสั่งดังนี้:
     - `CAM` : ตัดภาพเข้ากล้องหน้าผู้ประกาศ
     - `VO` (Voice Over) : ภาพพร้อมเสียงบรรยาย
     - `SOT` (Sound on Tape) : เสียงให้สัมภาษณ์
     - `VOSOT` : บรรยายแล้วเข้าภาพเสียงสัมภาษณ์ต่อเนื่อง
     - `PKG` (Package) : เทปรายงานข่าวสมบูรณ์ (สกู๊ปข่าว)
     - `OC` (On Camera) : จบแพ็กเกจแล้วกลับมาหน้าผู้ประกาศ
     - `CG` (Character Generator) : กราฟิกแสดงชื่อบุคคล/สถานที่
     - `FS` (Full Screen) : กราฟิกแสดงข้อมูลแบบเต็มหน้าจอ
     - `OTS` (Over The Shoulder) : กราฟิกประกอบข่าวข้างไหล่ผู้ประกาศ
     - `BUG` : โลโก้หรือสัญลักษณ์มุมจอ
     - `OUTCUE` : คำลงท้ายของวิดีโอ (จบด้วยคำว่า...)
     - `TRT` (Total Running Time) : ความยาวรวมทั้งหมดของตัววิดีโอ
     - `NATS` (Natural Sound) : ปล่อยเสียงสภาพแวดล้อมจริงให้ดังขึ้น
     - `DISS` (Dissolve) : เปลี่ยนภาพด้วยเทคนิคภาพจางซ้อนกัน
     - `LIVE` : ตัดเข้าสัญญาณสด
     - `PHONE` : สัญญาณโฟนอิน/สัมภาษณ์สดทางโทรศัพท์
     - `RAW` : พิมพ์ข้อความหรือคำสั่งรูปแบบอิสระ
   - **ฝั่งขวา (Anchor Text):** บทพูดสำหรับผู้ประกาศข่าวโดยเฉพาะ พร้อมการนับคำและเวลา
   - **คำสั่งเทคนิคด่วน (Inline Command Autocomplete):** รองรับการกดคีย์ `[` (วงเล็บเหลี่ยมเปิด) ในช่องพิมพ์ข้อความฝั่งขวา เพื่อเรียกเมนู Popup แนะนำคำสั่งให้ล็อกเป้าและแทรกลงในข้อความได้อัตโนมัติ รวดเร็วเหมือนระบบแชทสมัยใหม่

2. **เมนูนำทางและแถบเมทาดาตา (Top Navigation & Metadata)**
   - แถบเมนูหลัก (Story, Rundown, Assignment, Admin) พร้อม Dropdown ทันสมัย
   - ช่องกรอกผู้สื่อข่าว (Reporter) รองรับระบบ Autocomplete ค้นหารายชื่อจากฐานข้อมูล
   - เปลี่ยนแปลงสถานะงานได้ 4 ระดับ (DRAFT, READY, REVIEW, APPROVED)

3. **ค้นหาข่าวเก่าแบบพรีเมียม (Archive Search Sidebar) + ระบบเทียบเสียงพ้อง (Phonetic Search)**
   - หน้าค้นหาข่าวด้านข้าง (Sidebar) ที่แสดงผลข่าวเก่าในรูปแบบการ์ด (Cards) ที่สวยงามและอ่านง่ายมีป้ายบอกสถานะชัดเจน
   - **✨ ฟีเจอร์ใหม่:** ค้นหาข้ามการสะกดผิดด้วยเทคโนโลยี **Thai/English Soundex** ระบบจะแปลงค่า Keyword เป็นรหัสเสียงล่วงหน้า ทำให้ค้นหา "สมรักษ์" แล้วเจอได้แม้จะพิมพ์ค้นหาผิดว่า "สมรัก" หรือ "สรรมลักษณ์"

4. **พื้นที่ส่วนตัว "My Story" และ "My Bin" (Personal Workspace & Soft Delete)**
   - **หน้าต่าง My Story:** แถบหน้าต่างพิเศษสำหรับแสดงผลเฉพาะ "ข่าวที่คุณเป็นผู้เขียน" เข้าถึงได้จากเมนูด้านบน เพื่อความรวดเร็วในการติดตามงาน
   - **ระบบถังขยะส่วนตัว (My Bin):** กดย้ายข่าวลงถังขยะแบบคลีนๆ ข่าวที่ถูกทิ้งจะไม่ไปปะปนในหน้าค้นหาหลัก (Find Story)
   - ฟังก์ชันจัดการข่าว (Action Icons): สั่งโหลดข่าวเพื่อ View, Edit หรือสั่ง Del ส่งลงถังขยะด่วนได้รวดเร็วทันที หากสถานะของข่าวคุณยังเป็น DRAFT

5. **ระบบสกัดคำสำคัญและแจ้งเตือนอัตโนมัติ (Auto-Keyword Extraction & Status Validation)**
   - ทุกครั้งที่มีการ "เปลี่ยนสถานะข่าว" (เช่น จาก DRAFT สู่ READY) ระบบจะใช้ `Intl.Segmenter` ดึงข้อความไปประมวลผล กรอง Stop Words ขยะทิ้ง และหา Top 5 คำที่โผล่บ่อยที่สุดออกมา
   - แสดงหน้าต่าง Popup แจ้งเตือนเพื่อให้บรรณาธิการยืนยันคีย์เวิร์ดก่อนส่งเข้าระบบศูนย์กลาง ลดภาระคนทำงานและเพิ่มความแม่นยำให้ฐานข้อมูลข่าว

6. **ระบบนับคำและคำนวณเวลาอัจฉริยะ (Smart Word Count & Time Calculation)**
   - **เทคโนโลยีที่ใช้ตัดคำภาษาไทย (Thai Word Segmentation Technology):** ใช้มาตรฐาน Native Web API ที่ชื่อว่า **`Intl.Segmenter`** ซึ่งเป็นเอนจินของเบราว์เซอร์แยกแยะคำและพยางค์ภาษาไทยได้อย่างแม่นยำระดับ Native
   - **วิธีการคำนวณเวลา:** กำหนดมาตรฐานความเร็วในการอ่านข่าวของผู้ประกาศอยู่ที่ **3 คำ ต่อ 1 วินาที** (ปัดเศษขึ้นด้วย `Math.ceil()`) แปลงหน่วยเข้าแสดงผลแบบ `01:30` ทันทีบนหน้าจอ

7. **ระบบฐานข้อมูลแบบ Hybrid และ Versioning (Hybrid JSON Storage & Smart Auto-Save)**
   - **Hybrid Storage:** ถอดภาระข้อความหนักๆ ออกจาก SQLite โดยย้ายข้อมูล Script 2 คอลัมน์เต็มยศไปเก็บในโครงสร้างไฟล์ `.json` แทน ลดสภาวะ Database บวม
   - **Smart Versioning:** ทำการเก็บประวัติ `current_version` ไว้ในฐานข้อมูล
     - ถ้ากด **Save Story ด้วยมือ** ➡️ สร้างไฟล์อดีต Version ถัดไปทันที (`_v1`, `_v2`)
     - ถ้าเป็นการ **Auto-Save ทุกๆ 5 นาที** ➡️ อัปเดตทับไฟล์เดิม ไม่เพิ่มเลขเวอร์ชัน ไม่สร้างขยะในระบบ ลดปริมาณดิสก์เต็ม

8. **รูปแบบเอกสารพิมพ์ที่สมบูรณ์แบบ (Optimized Print Layout)**
   - รูปแบบปรับแต่งพิเศษสำหรับการพิมพ์ลงหน้ากระดาษ A4 เสมือนทำงานบน MS Word (พร้อมการจัดการ Margin ให้อัตโนมัติ)
   - โครงสร้าง UI (ปุ่มต่างๆ สีพื้นหลัง) จะถูกแปลงเป็นเอกสารขาว-ดำทั้งหมด เพื่อช่วยประหยัดหมึกพิมพ์
   - **✨ ระบบ Direct Print ทรงพลัง:** สั่งปริ้นท์แยกจากส่วนของหน้ากระดาษทำงานหลักได้รวดเร็วทันใจในเสี้ยววินาที ข้ามปุ่ม UI รกๆ แล้วจัดหน้าเข้า A4 แจ้งเตือนเสร็จปิดแท็บอัตโนมัติ

9. **ระบบจัดการคิวออกอากาศผังรายการ (Rundown Management Module)**
   - กระดานทำงาน (Dashboard) เรียงคิวอ่านข่าวด้วยระบบ Drag & Drop ลากปรับคิวสลับได้ลื่นไหล
   - การทำงานเชื่อมกันแบบเรียลไทม์ผ่านกลไก Polling (ไม่ต้อง Reload จอ) สลับออเดอร์ปุ๊บ หน้าจอทุกคนอัปเดตตามทันที
   - ระบบ Drop Story ตัดข่าวชั่วคราวแต่ยังคงโครงสร้างไว้ในกระดาน
   - ผูกระบบเป้าหมายเวลารายการ (Target TRT) คอยคำนวณบวกเวลาข่าวบนใบงานอัตโนมัติ พร้อมเด้งแถบการแจ้งเตือน "Red Alert" ทันทีหากเวลาเกินยอดผังรายการที่ขอไว้
   - พ่วงสิทธิ์สำหรับ "บก.กลาง (Main Editor)" ในการกด Lock Rundown เปลี่ยนใบงานทุกคนไปแสดงผลในตาราง Read-Only 

10. **ระบบบริหารจัดการชื่อรายการ (Program Master Data Admin)**
    - หน้าต่างเมนูผู้ดูแลระบบ (`admin.php`) ในการจัดการคลังข้อมูล "ชื่อรายการ" 
    - สามารถป้อนระยะเวลาเวลา (Duration) และ "จำนวนเบรกโฆษณา" ที่ต้องการให้แทรกอัตโนมัติ (Auto-inject Breaks) ผูกติดไว้ให้ทุกครั้งที่สร้างผังตาราง Rundown ใหม่ของรายการนั้นๆ

11. **ดีไซน์พรีเมียมเรืองแสง (Premium Glassmorphism & UI)**
    - ปรับแก้ระบบ Input ตามจุดต่างๆ ทั่วทุกมุมเว็บไซต์ให้กลายเป็นแบบมีมิติ (Inset shadow), โทนกระจกละลาย (Backdrop-filter blur) 
    - เสริมความหรูหราให้เมนู Drop-down, การเด้งสู้เมาส์ของปุ่ม และแทรกสัญลักษณ์จาก FontAwesome เพื่อประหยัดพื้นที่ทำงานในส่วน Rundown

12. **ระบบตรวจสอบสิทธิ์ระดับสูง (Role-Based Access Control - RBAC)**
    - **นักข่าว (Reporter):** สิทธิเบื้องต้น สร้างและพิมพ์ได้แค่ข่าวสังกัดตน แต่อนุมัติ (APPROVED) ข่าวไม่ได้ และไม่มีสิทธิ์เข้าถึงหน้ารายการ Rundown ให้ดูเฉพาะงานของตนเอง
    - **บก. สังกัด (Editor):** สิทธิ์แก้ไขระดับกลาง อนุมัติ (APPROVED) ข่าวได้เฉพาะข่าวสังกัดตนเองเท่านั้น และมีสิทธิ์เข้าถึงรับชมหน้ารายการ Rundown อัพเดทสดได้
    - **นักรีไรต์ (Rewriter):** ตำแหน่งพิเศษสำหรับแก้ไขสำนวน หยิบข่าวมาแก้ได้อิสระ **ทุกสังกัด** (แต่อย่างไรก็ Approved ข่าวไม่ได้ และไม่มีสิทธิ์เข้าถึงหน้ารายการ Rundown)
    - **บก. กลาง (Main Editor):** สิทธิ์ God-mode ควบคุมได้ทุกสิ่ง จัดตั้งตารางการอ่านข่าว จัดการ Master Data สร้างหน้า Rundown ลบ แก้ไข อนุมัติ ทุุกข่าวทุกกระทรวง

13. **ระบบป้องกันการแก้ไขซ้อนทับ (Story Lock & Collision Prevention)**
    - ผูกระบบล็อกตั๋วเข้าทำงานกับ `employee_id` ทันทีที่มีผู้เปิดอ่านหรือแก้ไขบทข่าว
    - ป้องกันปัญหาเขียนทับกัน (Race Condition) หากผู้อื่นพยายามเปิดข่าวเดียวกัน UI จะถูกปรับให้เป็นโหมด `Read Only` ทันที (ปุ่มเซฟล็อก/ปิดการทำงาน Input)
    - ระบบส่งสัญญาณชีพจร (Heartbeat) ต่ออายุการ Lock อัตโนมัติทุกๆ 2 นาทีตราบที่ผู้เขียนยังคงใช้งานข่าวอยู่
    - ปลดล็อกอัตโนมัติรวดเร็วผ่าน `navigator.sendBeacon` พร้อมแฝง CSRF Token ป้องกันการถูกจำลองคำสั่งยิงปลดล็อกทันทีที่มีคนเปลี่ยนหน้าต่าง

14. **ระบบรักษาความปลอดภัยครบวงจร (Security & Protection)**
    - โครงสร้างป้องกัน Directory Traversal ล็อกไฟล์ Database และประวัติ JSON ผ่าน `.htaccess`
    - ระบบต่อต้านการเจาะช่องโหว่ XSS Injection เต็มรูปแบบ ทั้งในรูปแบบการแสดงผล DOM Elements, Modals และ Inline Script JSON Encodings
    - ระบบการเคลียร์บัญชีและระงับ Session Cookies ทิ้งทันทีเมื่อกดออกจากระบบ ป้องกัน Session Fixation อย่างรัดกุม

15. **ระบบกระดานควบคุมข้อมูลและสถิติ (Admin Dashboard & Analytics)**
    - แดชบอร์ดแบบเรียลไทม์ (`dashboard.php`) วิเคราะห์และแสดงกราฟ (Chart.js) สถานะข่าว (Story Status)
    - ติดตามและประเมินภาระงานของแต่ละแผนก (Department Workload) ดูภาพรวมการทำงานในองค์กรแบบรวดเร็ว

16. **ระบบหมายข่าวและจองคิวอุปกรณ์ (Assignment & Equipment Booking)**
    - **หมายข่าว (Assignment):** ระบบสั่งจ่ายหมายข่าวให้กับผู้สื่อข่าวและตากล้อง พร้อมหน้าต่างสั่งปริ้นท์ใบหมายงาน (`print_assignment.php`)
    - **จัดการอุปกรณ์ (Equipment):** จัดการคลังอุปกรณ์และติดตามสถานะการถูกจอง/ใช้งาน สำหรับการออกกอง

## 🛠 เทคโนโลยีที่ใช้งาน (Tech Stack)

- **Frontend:** HTML5, Vanilla JavaScript (ES6 Modules), CSS3 (Modern UI), Chart.js (Dashboard Analytics) + SweetAlert2
- **Backend:** PHP 8+ (Phonetic Logic, API & Session Authentication)
- **Database / File System:** SQLite (อยู่โฟลเดอร์ `database/`) + JSON Data Archives (`data/stories/`)

## 🚀 การติดตั้งและใช้งานระบบ (Installation & Setup)

1. นำโฟลเดอร์รันบน Web Server ท้องถิ่น (เช่น XAMPP, Laragon, MAMP หรือ `php -S localhost:80`)
2. เปิดเบราว์เซอร์และเข้าไปที่โฮสต์ที่ตั้งไว้ เช่น `http://localhost/newsroom`
3. ในครั้งแรกที่รัน ระบบจะสร้างไฟล์ `newsroom.sqlite` และสร้างฐานข้อมูลพร้อมโฟลเดอร์ `.json` ให้อัตโนมัติ

**บัญชีสำหรับทดสอบที่มาพร้อมระบบ (Default Credentials):**
*(รหัสผ่านสำหรับทุกบัญชีคือ `1234`)*
- `admin` (บกกลาง / การเมือง) : สิทธิ์สูงสุด จัดการ/Approve ได้ทุกอย่าง
- `editor1` (บก / สังคม) : แก้ไขและ Approve ข่าวสังคม
- `reporter1` (นักข่าว / สังคม) : สร้างและพิมพ์ข่าวให้สังกัดสังคม (Approve ไม่ได้)
- `rewrite1` (rewriter / กีฬา) : ดึงข่าวข้ามสังกัดมาแก้เนื้อหาได้ทั้งหมด (Approve ไม่ได้)

## 📂 โครงสร้างไฟล์ใหม่ตามสถาปัตยกรรม (Project Structure)

- `index.php` - หน้าหลักสำหรับเขียนบทข่าว (Main Editor UI)
- `rundown.php` - แดชบอร์ดเรียงคิวลำดับข่าว (Rundown Management Engine)
- `dashboard.php` - แดชบอร์ดแสดงผลสถิติและกราฟวิเคราะห์ (Admin Analytics)
- `assignment.php` - ระบบจัดการใบหมายข่าวและมอบหมายงานให้ทีม
- `equipment.php` - ระบบคลังจัดการอุปกรณ์และแบบฟอร์มการเบิกจ่าย
- `admin.php` - หน้าตั้งค่ารายการและโครงสร้าง Rundown (Master Data Control)
- `departments.php` / `users.php` - ระบบจัดการข้อมูลโครงสร้างแผนกและสิทธิผู้ใช้งานองค์กร
- `top_menu.php` - ไฟล์โมดูลของแท็บเมนูการเข้าถึงหน้าต่างต่างๆ ระบบค้นหาข่าวและสถานะผู้ใช้ 
- `print_story.php` - สคริปต์สกัดข้อมูลพิมพ์บทข่าวแบบ A4 ขาว-ดำ
- `print_assignment.php` - สคริปต์สั่งพิมพ์ใบหมายงาน
- `login.php` / `logout.php` - ระบบป้องกันและตรวจสอบ Session
- `api.php` - เอนจินหลังบ้านทำหน้าที่รับ JSON Payload, ขับเคลื่อนฐานข้อมูล, ลอจิก Soundex
- `db.php` - เอนจินตั้งต้น SQLite 
- `js/` - โมดูลแยกการทำงาน JavaScript ตามฟังก์ชัน (ES6) 
- `style.css` - สไตล์ดีไซน์หลักกระจกแบบพรีเมียม (Premium Theme)
- `database/` - ซุ้มเก็บไฟล์ SQLite (`newsroom.sqlite`)
- `data/stories/` - ซุ้มเก็บประวัติข้อความและเวอร์ชัน JSON
