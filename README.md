# AP ENPS Script Editor MVP 📰

ระบบเขียนและจัดเตรียมบทข่าว (Newsroom Script Editor) ที่พัฒนาตามมาตรฐานการทำงานของระบบข่าวระดับโลก (คล้าย ENPS หรือ iNews) ถูกออกแบบให้ใช้งานง่าย รวดเร็ว และรองรับการพิมพ์บทข่าวสำหรับให้ทีมงานและผู้ประกาศข่าวอ่านได้อย่างมีประสิทธิภาพ

## 🌟 ฟีเจอร์หลัก (Key Features)

1. **ระบบบทข่าว 2 คอลัมน์ (Dual-Column Editor)**
   - **ฝั่งซ้าย (Production Cues):** คำสั่งเทคนิคสำหรับทีมผลิต (Director, CG, Audio) ในรูปแบบบล็อกข้อมูล (Structured Blocks) มี Dropdown ให้เลือกประเภท (CAM, CG, VO, SOT, LIVE ฯลฯ)
   - **ฝั่งขวา (Anchor Text):** บทพูดสำหรับผู้ประกาศข่าวโดยเฉพาะ พร้อมการนับคำและเวลา

2. **ระบบนับคำแบบอัจฉริยะภาษาไทย (Smart Thai NLP Word Count)**
   - คำนวณความยาวของข่าวและตัดคำภาษาไทยได้อย่างแม่นยำด้วยเทคโนโลยี `Intl.Segmenter` ช่วยคำนวณเวลาการอ่านที่อิงกับความเร็วผู้ประกาศข่าวตามจริง (3 คำ / วินาที)

3. **ระบบ Auto-Save อัตโนมัติ (Intelligent Save Mechanism)**
   - ตัวระบบจะทำการบันทึกข้อมูลแบบอัตโนมัติ (Auto-save) ทุกๆ 5 นาที ไม่รบกวนการทำงาน
   - ทุกครั้งที่กดสั่งพิมพ์ (Print) ระบบจะ Save ก่อนเสมอเพื่อความแม่นยำของข้อมูล

4. **รูปแบบเอกสารพร้อมพิมพ์ (Print / PDF View)**
   - รูปแบบปรับแต่งพิเศษ (A4 Print Layout) เมื่อสั่ง Print โครงสร้าง UI (ปุ่มต่างๆ สีพื้นหลัง) จะโดนซ่อนทั้งหมด และแสดงรายละเอียดของ Slug, ตำแหน่งต่างๆ บนหัวกระดาษเป็นสีดำบนพื้นหลังขาว เพื่อประหยัดหมึกพิมพ์

5. **ระบบจัดการผู้ใช้และความปลอดภัย (Authentication System)**
   - มีการแยกโครงสร้างฐานข้อมูลระหว่าง Roles (ตำแหน่งหลัก) และ Departments (สังกัดงาน) ชัดเจน เพื่อความยืดหยุ่นในการจัดการ
   - มีระบบจดจำล็อกอิน (Session) และล็อกพาสเวิร์ด (Bcrypt) แบบปลอดภัย

6. **ระบบกำหนดสิทธิ์การเข้าถึง (Role-Based Access Control - RBAC)**
   - **นักข่าว (Reporter):** รับหน้าที่เขียนข่าวเบื้องต้น ไม่สามารถเปลี่ยนสถานะเป็น APPROVED ได้ สร้างและแก้ไขข่าวได้เฉพาะในสังกัดตนเองเท่านั้น
   - **บก. สังกัด (Editor):** สามารถตรวจทาน แก้ไข และเปลี่ยนสถานะตัวข่าวเป็น APPROVED หรือ REVIEW ได้ **เฉพาะข่าวภายใต้สังกัดของตนเองเท่านั้น** (ไม่สามาถ Approve ข้ามสายงานได้)
   - **บก. กลาง (Main Editor):** มีสิทธิ์การแก้ไขและอนุมัติ (APPROVE) รวมไปถึงย้ายสังกัดของบทข่าวได้ **ทุกข่าว ทุกสังกัด** ในระบบ
   - **นักรีไรต์ (Rewriter):** ตำแหน่งพิเศษสำหรับแก้ไขสำนวนภาษา มีสิทธิ์เข้าแก้ไขเนื้อหาข่าวได้ **ทุกข่าวและทุกสังกัด** แต่ไม่สามารถกดยืนยัน (APPROVE) บทข่าวในขั้นสุดท้ายได้

## 🛠 เทคโนโลยีที่ใช้งาน (Tech Stack)

- **Frontend:** HTML5, Vanilla JavaScript, CSS3 (Modern UI, Dark Mode Default 🌙)
- **Backend:** PHP 8+ (API & Session Authentication)
- **Database:** SQLite (อยู่ในโฟลเดอร์ `database/newsroom.sqlite` - สร้างขึ้นเองอัตโนมัติ)

---

## 🚀 การติดตั้งและใช้งานระบบ (Installation & Setup)

1. นำโฟลเดอร์รันบน Web Server ท้องถิ่น (เช่น XAMPP, Laragon, MAMP หรือ `php -S localhost:80`)
2. เปิดเบราว์เซอร์และเข้าไปที่โฮสต์ที่ตั้งไว้ เช่น `http://localhost/newsroom`
3. ในครั้งแรกที่รัน ระบบจะสร้างไฟล์ `newsroom.sqlite` และสร้างฐานข้อมูลให้อัตโนมัติ

**บัญชีสำหรับทดสอบที่มาพร้อมระบบ (Default Credentials):**
*(รหัสผ่านสำหรับทุกบัญชีคือ `1234`)*
- `admin` (บกกลาง / การเมือง) : สิทธิ์สูงสุด จัดการ/Approve ได้ทุกอย่าง
- `editor1` (บก / สังคม) : แก้ไขและ Approve ข่าวสังคม
- `reporter1` (นักข่าว / สังคม) : สร้างและพิมพ์ข่าวให้สังกัดสังคม (Approve ไม่ได้)
- `rewrite1` (rewriter / กีฬา) : ดึงข่าวข้ามสังกัดมาแก้เนื้อหาได้ทั้งหมด (Approve ไม่ได้)

## 📂 โครงสร้างไฟล์ (Project Structure)

- `index.php` - หน้าหลักสำหรับเขียนบทข่าว (Main Editor UI)
- `login.php` - หน้าเข้าสู่ระบบ (Login UI & Authentication Logic)
- `logout.php` - สคริปต์สำหรับออกจากระบบ
- `api.php` - สคริปต์หลังบ้านทำหน้าที่รับข้อมูล JSON และสื่อสารกับฐานข้อมูล
- `db.php` - ระบบจัดการฐานข้อมูล SQLite (สร้างโครงสร้าง / เพิ่มบัญชีเริ่มต้น)
- `app.js` - สคริปต์หน้าบ้านสำหรับทำ Auto-save, UI Interaction และการนับเวลา (NLP)
- `style.css` - สไตล์หลักของเว็บไซต์ (UI & Print Layout Media Queries)
- `database/` - โฟลเดอร์สำหรับเก็บไฟล์ฐานข้อมูล (`newsroom.sqlite`)
